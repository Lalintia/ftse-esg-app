FTSE ESG Analyzer

AI-powered ESG analysis tool that evaluates company websites against FTSE Russell sustainability standards

Internal Sales Tool • Version 2.0 • March 2026
1 Project Overview

โปรเจกต์นี้ทำอะไร ทำเพื่อใคร และทำไมถึงต้องทำ

จุดประสงค์

เครื่องมือสำหรับ ทีม Sale ใช้วิเคราะห์เว็บไซต์ของลูกค้า เพื่อดูว่าบริษัทมีข้อมูล ESG (สิ่งแวดล้อม สังคม ธรรมาภิบาล) ครบถ้วนแค่ไหน ตามมาตรฐาน FTSE Russell


วิธีใช้: Sale ใส่ URL เว็บไซต์ลูกค้า → ระบบวิเคราะห์อัตโนมัติ → ได้คะแนน ESG + จุดที่ต้องปรับปรุง → Sale นำไปเสนอลูกค้า


Business Value: ช่วย Sale pitch ลูกค้าว่า "เว็บไซต์ของคุณขาดข้อมูล ESG ตรงนี้ เราช่วยทำได้" — เปลี่ยน ESG gap เป็นโอกาสขายบริการ

3 User Flow — ผู้ใช้ทำอะไรบ้าง

ขั้นตอนการใช้งานจากมุม Sale team

1. กรอก URL + เลือก Industry
Sale ใส่ URL เว็บไซต์หลักของลูกค้าเพียง URL เดียว เช่น ptgenergy.co.th — ระบบจะค้นหาเว็บที่เกี่ยวข้องเอง เช่น investor.ptgenergy.co.th หรือ subdomain อื่นๆ (IR, Sustainability, Corporate) โดยอัตโนมัติ ไม่ต้องใส่ทีละเว็บ
Industry: ถ้ารู้หมวดธุรกิจก็เลือกได้เลย (เช่น Oil, Gas & Energy) ถ้าไม่รู้ก็เลือก Auto-detect แล้ว AI จะตรวจจากเนื้อหาเว็บให้
หน้าเว็บ → esg.ohmai.me
2. ระบบ Crawl เว็บไซต์
AI เข้าไปอ่านเนื้อหาทั้งเว็บไซต์ ทุกหน้าที่เกี่ยวข้องกับ ESG รวมถึง subdomain และดาวน์โหลด PDF (Annual Report, Sustainability Report) อัตโนมัติ
Playwright + pdfplumber • ~2-3 นาที
3. AI วิเคราะห์ ESG Indicators
AI วิเคราะห์เนื้อหาเทียบกับ FTSE Russell ESG indicators ที่เกี่ยวข้องกับหมวดธุรกิจนั้นๆ ครอบคลุม themes ด้าน Environment, Social และ Governance
OpenAI GPT-4.1-mini • ~3-4 นาที
4. คำนวณคะแนน ESG
คิดคะแนน 0-5 ต่อ theme ตาม FTSE methodology (exposure-weighted) แล้วรวมเป็นคะแนน E, S, G และ Overall
Scoring Engine • < 1 วินาที
5. แสดงผล Dashboard + Sitemap
Sale เห็นคะแนนรวม, คะแนนรายธีม, รายละเอียด indicator ที่เจอ/ไม่เจอ, และคำแนะนำว่าควรสร้างหน้าเว็บอะไรเพิ่ม
Dashboard → พร้อมส่งลูกค้า
4 Analysis Pipeline — ระบบทำงานอย่างไร

ขั้นตอนการทำงานภายในของ backend ทั้งหมด

1 Sitemap Scan ค้นหา sitemap.xml
2 Web Crawl Playwright 50 หน้า + subdomain
3 PDF Discovery TOC scan → อ่านเฉพาะหน้า ESG
4 Auto-detect ตรวจหมวดธุรกิจ (ICB)
5 Filter เลือก indicators ที่เกี่ยวข้อง
6 Round 1 วิเคราะห์จาก web content
7 Round 2 เติม gaps จาก PDF
8 Score คำนวณ ESG 0-5
9 Sitemap แนะนำหน้าเว็บที่ควรเพิ่ม
Smart PDF Reading

PDF รายงานประจำปีอาจมี 300+ หน้า แต่ระบบไม่อ่านทุกหน้า — สแกนสารบัญก่อน แล้วอ่านเฉพาะหน้าที่เกี่ยวข้องกับ ESG (governance, sustainability, safety, water ฯลฯ) ทั้งภาษาไทยและอังกฤษ ประหยัดเวลาและ token

8 Security Measures

มาตรการรักษาความปลอดภัยที่ทำไว้

🛡
SSRF Protection Block private IPs (127.x, 10.x, 169.254.x) ป้องกันการเข้าถึง AWS metadata
⏱
Rate Limiting 5 requests/นาที สำหรับ analysis, 30 requests/นาที สำหรับ API อื่น
🔒
Security Headers X-Content-Type-Options, X-Frame-Options, XSS-Protection, HSTS
👤
Non-root Docker Backend container รันเป็น appuser (uid 1001) ไม่ใช่ root
🌐
CORS Restricted อนุญาตเฉพาะ esg.ohmai.me และ localhost:3000
📊
Query Limits Pagination จำกัด max 100 records ต่อ request
📝
XML DoS Protection Sitemap XML จำกัด 10MB ป้องกัน billion laughs attack
⚠
Pending: Authentication ยังไม่มี login — ใครมี URL ก็เข้าได้ (planned for next phase)
